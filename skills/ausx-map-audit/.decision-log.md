# Decision Log: ausx-map-audit

## 2026-05-20

- Classified as a module-based simple workflow because the core outcome is a bounded audit report, not an agent persona or long-running memory relationship.
- Kept the workflow read-only so map validation cannot accidentally modify authored feature archives or living ledgers.
- Chose Markdown with an embedded JSON summary instead of a separate required script. The LLM can perform judgment on inconsistencies while preserving a stable output contract for future UI ingestion.
- Made projection rebuild readiness explicit because downstream Auspex reporting should not treat generated maps as source truth.
