# Decision Log: ausx-salmon-impact

## 2026-05-20

- Classified as a module-based simple workflow because it produces a bounded consistency report from explicit upstream-impact signals.
- Defined Salmon signals broadly enough to catch both metadata and prose notes without requiring a custom parser in v1.
- Kept topology changes out of this workflow. Salmon can recommend topology review, but `ausx-topology-design` owns those decisions.
- Required direct evidence for blocking status so advisory cleanup does not halt teams unnecessarily.

## 2026-05-21

- Expanded Salmon traversal to check both upstream parentage and downstream or sibling dependents named by links, relationships, shared contracts, or source breadcrumbs.
- Added JSON fields for reviewed upstream/downstream nodes and confidence so reporting snapshots can preserve impact scope.
